# Stock Lab Framework

## 目标

建立一个可扩展、可解释、面向多市场的 **股票研究 + 模拟交易 + 看板可视化框架**。

## 设计原则

- **框架优先**：先提供稳定可运行的结构，再按使用者需求注入交易逻辑
- **状态统一**：研究、账户、持仓、订单、风控与解释信息都归入统一状态文件
- **策略可替换**：交易风格、候选池、评分器与退出规则都应可独立替换
- **本地优先**：默认面向本地部署与本地数据控制
- **多市场兼容**：A / HK / US 可按各自市场节奏独立运行

## 系统主流程

1. 拉取行情和外部数据
2. 更新账户、持仓与风险暴露
3. 执行纪律与风控检查
4. 生成候选、观察项与解释信息
5. 输出统一状态文件
6. 前端看板读取状态并渲染

## 核心模块

- `core/config.py`：配置与模板默认值
- `core/market_data.py`：多数据源接入层
- `core/strategy.py`：候选构建 / 评分 / 审议 / 进场
- `core/discipline.py`：止损 / 止盈 / 风控 / 退出
- `core/state_builder.py`：状态聚合与 schema 输出
- `core/stock_analysis.py`：个股分析与异动识别逻辑
- `static/`：桌面端与移动端 UI

## 可扩展点

使用者可以基于这个框架自行定义：

- watchlist 来源
- 主题分类
- 候选评分器
- 风险阈值
- 调仓与退出规则
- 个股分析口径
- 前端展示方式

## 建议使用顺序

1. 先运行空模板，确认环境与页面正常
2. 再填写 `TRADING_STRATEGY.md`
3. 再生成自己的 `config.json`
4. 再改 `core/strategy.py` 与 `core/discipline.py`
5. 最后接入调度、自动刷新或更多数据源
