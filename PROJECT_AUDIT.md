# Framework Audit Notes

## 当前公开仓库的定位

这是一个用于本地部署的 **多市场股票研究 / 模拟交易 / 看板框架**。

它的目标不是提供预设交易结论，而是提供一套可以被使用者或使用者的 agent 继续定制的工程骨架。

## 当前保留的部分

- `trading_engine.py`：状态生成主入口
- `dashboard_server.py`：轻量 HTTP 服务
- `core/`：配置、数据、风控、策略、状态拼装
- `static/`：桌面端 + 移动端看板 UI
- `tests/`：核心行为回归测试
- `.env.example` / `config.example.json`：模板配置
- `TRADING_STRATEGY.md`：空白策略模板

## 当前模板特征

- 默认 `watchlists` 为空
- 默认不提交本地配置与运行态文件
- 默认不写入现成交易风格内容
- 默认由使用者自行补充市场、板块、风控和频率要求

## 设计上的核心抽象

### 1. 状态中心

后端把研究、仓位、订单、纪律、风险、解释信息统一生成到一个状态文件；前端只负责渲染。

### 2. 策略外置

策略文档、配置文件和 Python 逻辑分层存在，便于 agent 读取文档后再把规则落到代码里。

### 3. 多市场会话

A / HK / US 按各自市场时区切换盘前、盘中、盘后阶段，而不是共享一套硬编码时钟。

### 4. 本地可控

默认面向本地运行、本地配置、本地持有数据源凭证，降低共享仓库时的泄漏风险。

## 建议的二次开发顺序

1. 克隆仓库并跑通测试
2. 复制 `config.example.json` 为 `config.json`
3. 填写 `TRADING_STRATEGY.md`
4. 修改 `core/strategy.py` / `core/discipline.py`
5. 启动看板并根据自己的使用场景调整 UI
