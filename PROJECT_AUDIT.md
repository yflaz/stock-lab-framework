# Public Framework Audit Notes

这个仓库公开版本的目标是：

- 保留多市场研究 / 模拟交易框架
- 保留状态驱动看板架构
- 保留可扩展的数据源与调度接口
- 不公开作者个人持仓、观察名单、复盘记录和私有交易模板

## 公开版保留的部分

- `trading_engine.py`：状态生成主入口
- `dashboard_server.py`：轻量 HTTP 服务
- `core/`：配置、数据、风控、策略、状态拼装
- `static/`：桌面端 + 移动端看板 UI
- `tests/`：核心行为回归测试
- `.env.example` / `config.example.json`：模板配置

## 公开版刻意不提交的部分

- `config.json`
- `.env`
- `simulation_state.json`
- `sector_summary.json`
- `strategy_journal/`
- 运行时备份与缓存文件

## 设计上的核心抽象

### 1. 状态中心

后端把研究、仓位、订单、纪律、风险、解释信息统一生成到一个状态文件；前端只负责渲染。

### 2. 策略可替换

这个仓库把“框架”和“具体交易偏好”分离：

- 框架：账户、时段、风控、状态、界面
- 策略：候选池、评分器、放行条件、退出风格

### 3. 多市场会话

A / HK / US 不是用一套硬编码时间强行共用，而是按各自市场时区切换盘前、盘中、盘后阶段。

### 4. 纪律优先

无论你最终接什么策略，仓位纪律都应该独立于主观观点执行。

## 建议的二次开发顺序

1. 先跑通空模板配置
2. 再填自己的 watchlist / 数据源
3. 再替换 `core/strategy.py`
4. 最后再接自动调度或实盘接口
