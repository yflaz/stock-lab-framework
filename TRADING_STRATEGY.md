# Framework Strategy Surface

这个文件只描述 **框架层的可配置面**，不提供作者个人的选股结论。

## 框架内置的动作类型

- `direct_follow`: 满足你自己的放行条件后，允许在纸面账户里模拟建仓
- `watch_pullback`: 逻辑未破坏，但等待更好的价格位置
- `watch_confirm`: 等待量价、事件或板块确认
- `observe_only`: 仅保留监控，不进入交易
- `avoid_for_now`: 当前赔率或风险结构不合适

## 你可以自定义的策略层

### 1. 候选池来源

在 `core/config.py` / `config.json` 中决定候选来自：

- 固定 watchlist
- 板块轮动列表
- 事件驱动列表
- 你自己的选股脚本输出

### 2. 候选评分逻辑

`core/strategy.py` 负责：

- 主题/行业偏好加权
- 动量或回撤状态判定
- 数据质量检查
- 资金和仓位约束
- 将候选映射到动作类型

### 3. 风控与退出规则

`core/discipline.py` 负责：

- 止损
- 分批止盈
- 移动止盈
- 时间止损
- 集中度控制
- 波动异常复核

### 4. 多市场交易时段

`core/utils.py` 与调度脚本负责：

- A / HK / US 各自按本地市场时区切换会话
- 非交易日不触发分析或执行
- 只在命中关键时段时更新状态

## 推荐的公开使用方式

如果你要基于这个仓库做自己的版本，建议：

1. 保留框架、看板和状态 schema
2. 用自己的 watchlist 和主题分类覆盖默认值
3. 用自己的入场 / 出场规则替换评分逻辑
4. 不把真实持仓、复盘笔记、私有 API key 提交到仓库
